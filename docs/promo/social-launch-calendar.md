# Social Launch Calendar

Goal: gain technical traction for ComfyUI Node Builder through a staggered 7-day launch focused on GitHub stars, useful node ideas, and feedback from ComfyUI node authors.

Primary CTA: star the GitHub repo.

Secondary CTA: open a node idea issue.

Primary link: https://github.com/caoool/comfyui-node-canvas

Secondary link: https://caoool.github.io/comfyui-node-canvas/

Feedback issue: https://github.com/caoool/comfyui-node-canvas/issues/2

## Day 0: Prep

- Export `demo-ai-node-workflow.gif` to `docs/promo/demo-ai-node-workflow.mp4` when `ffmpeg` is available.
- Export `docs/promo/github-social-preview.svg` to PNG for GitHub social preview if needed.
- Set the GitHub repo social preview image manually in GitHub settings.
- Pin the GitHub repo or landing page on social profiles for launch week.
- Keep one browser tab open for GitHub issues so node ideas can be captured quickly.

## Day 1: X/Twitter Main Launch

- Post the main X launch copy from `docs/promo/social-posts-x.md`.
- Attach the MP4 if available; otherwise attach `demo-ai-node-workflow.gif`.
- Add two thread replies: feature list and example AI prompt.
- Reply to every technical comment for the first 2 hours.

## Day 2: Reddit Technical Post

- Post one value-first Reddit post using `docs/promo/social-posts-reddit-hn.md`.
- Start with `r/StableDiffusion` or a ComfyUI-focused subreddit only if the current community rules allow it.
- Do not cross-post the same link to multiple subreddits on the same day.
- Ask for node-author feedback instead of only asking for stars.

## Day 3: GitHub Community Push

- Use GitHub issue #2: `Node ideas wanted: what ComfyUI nodes should the AI Builder support?`.
- Add 3 seed node ideas if no one has commented yet.
- Share the issue link in replies where people ask for examples.

## Day 4: Hacker News Show HN

- Submit the Show HN title and first comment from `docs/promo/social-posts-reddit-hn.md`.
- Stay available for the first 2-3 hours.
- Answer with implementation detail: local helper server, deploy gating, visible generated files, validation, and ComfyUI workflow testing.

## Day 5: X/Twitter Use-Case Follow-Up

- Post the prompt-enhancer node example from `docs/promo/social-posts-x.md`.
- Link to GitHub and the node idea issue.
- Ask people what node they would want AI to build next.

## Day 6: Direct Outreach

- Search for recent posts about ComfyUI custom nodes, Python node boilerplate, and ComfyUI workflow utilities.
- Reply only when relevant; do not paste the same link everywhere.
- Ask for workflow feedback from custom-node authors.

## Day 7: Recap And Poll

- Post the recap/poll copy from `docs/promo/social-posts-x.md`.
- Summarize any new issues or feature requests.
- Convert useful comments into GitHub issues.

## Tracking

Record after each post:

- date and channel
- post URL
- stars before and after
- landing page visits if available
- comments with useful feature requests
- node ideas opened
- follow-up action
